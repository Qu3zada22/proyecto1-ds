# Progreso de implementación: data-pipeline-bootstrap

## Estado

Estado actualizado con remediación final 4R y consistencia documental: además de las unidades PR 1, PR 2 y PR 3 ya completadas, se corrigieron los bloqueadores finales de reproducibilidad/corrección y warnings de documentación. La consolidación rechaza CSV crudos ragged antes de escribir salida; adquisición cae a manual cuando MINEDUC devuelve HTML/CSV inválido si existen CSV manuales; el CLI de consolidación restringe `--raw-dir` y `--manifest` a `data/raw`; diagnóstico reporta fallos de restauración con backups preservados; la documentación usa comandos portables `uv run ...`, aclara que la recaptura HTML de ejemplo es por departamento y alinea planificación con la fuente real HTML `html-form`. El alcance de limpieza, deduplicación parcial, normalización categórica y dataset final permanece diferido a fases posteriores.

Parcial. La unidad PR 1 implementó y remedió la base Python, fixtures, manifest y adquisición trazable. Los hallazgos críticos de cobertura CLI, justificación de alcance faltante y error no controlado para archivos manuales ausentes quedaron corregidos. También se atendieron los riesgos factibles de PR 1: HTTPS obligatorio para MINEDUC, restricción pública de `--raw-dir`, merge determinístico del manifest, rechazo de descargas vacías/HTML en la ruta CSV, escritura temporal antes de reemplazo confirmado, limpieza de `*.egg-info/` y rollback seguro cuando una sobrescritura confirmada falla al persistir el manifest. En la continuación de adquisición se investigó la página real de MINEDUC, no se encontró punto final CSV directo, se preservaron 23 artefactos HTML oficiales de resultados de `NIVEL ESCOLAR: DIVERSIFICADO` en `data/raw/` con manifest trazable, y antes de PR 2 se endureció la confiabilidad de adquisición: `manifest.json` se escribe atómicamente, el flujo de sobrescritura restaura también el manifest previo si la persistencia falla, el POST WebForms conserva `__VIEWSTATE`/`__EVENTVALIDATION`, el CLI permite recaptura HTML por departamento y la documentación reconoce el botón `Exportar a Excel` pendiente de investigación. La remediación R9 corrige el caso crítico de preparación del backup del manifest: si esa preparación falla, no se reemplaza el crudo. La remediación R10 corrige el crítico restante de restauración: el rollback ya no elimina el artefacto actual antes de restaurar desde backup, preserva el backup que no pudo restaurarse para recuperación manual y levanta `AcquisitionError` con contexto del destino, backup y causa. En la enmienda de planificación se actualizó la especificación, el diseño y las tareas para que PR 2 pueda operar legalmente sobre CSV crudos o HTML oficiales preservados, extrayendo tablas a `data/interim/establecimientos_diversificado_raw_unificado.csv` como consolidación estructural sin limpieza. La unidad PR 2 queda implementada: consolidación cruda acepta CSV y HTML `html-form`, compara encabezados normalizados solo para detectar compatibilidad, agrega `archivo_origen` y `departamento_origen`, bloquea incompatibilidades o HTML sin tabla esperada y generó el CSV intermedio real desde los 23 HTML oficiales preservados.
